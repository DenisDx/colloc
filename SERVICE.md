# Colloc System Service

## Overview

The Colloc system service manages auxiliary host-side processes required by the Colloc stack, specifically the **restart hook listener** that enables the "Restart system" button in the Web UI.

## Architecture

```
┌─────────────────────────────────────────────────────┐
│  Web UI (frontend)                                  │
│  - "Restart system" button                          │
└────────────────────┬────────────────────────────────┘
                     │ Click
┌────────────────────▼────────────────────────────────┐
│  webui-backend (container)                          │
│  - POST /api/system-reset                           │
│  - Sends: POST to http://host.docker.internal:9999 │
└────────────────────┬────────────────────────────────┘
                     │ HTTP POST
┌────────────────────▼────────────────────────────────┐
│  Host Machine (colloc_service.sh)                   │
│  - Listens on 127.0.0.1:9999                        │
│  - Handler: /restart-services                       │
│  - Executes: docker compose restart                 │
└────────────────────┬────────────────────────────────┘
                     │ Docker API
┌────────────────────▼────────────────────────────────┐
│  Docker Daemon                                      │
│  - Restarts all running services (profile core)     │
└─────────────────────────────────────────────────────┘
```

## Components

### 1. `colloc_service.sh`

**Location:** Project root  
**Purpose:** Single entry point for all host-side auxiliary services  
**Executable:** Yes, made executable by `install.sh`

Responsibilities:
- Start the restart hook listener in background
- Manage process lifecycle (SIGTERM, SIGINT)
- Log to `logs/service.log` and journal (if systemd)

### 2. `scripts/restart-service-hook.py`

**Language:** Python 3  
**Purpose:** HTTP server that handles container-triggered restart requests  
**Port:** 9999 (localhost only for security)

Features:
- Listens on `127.0.0.1:9999` (not exposed externally)
- Endpoint: `POST /restart-services` accepts JSON body
- Executes: `docker compose --profile core restart`
- Returns: JSON response with status/message
- Logs: `logs/restart-hook.log` and stdout/stderr

Security:
- No Docker socket mounting (no privilege escalation)
- Runs with regular user privileges (no sudo)
- Restricted to localhost via `host.docker.internal` from container

### 3. `colloc.service`

**Type:** systemd user/system service  
**Generated by:** `install.sh` (not committed to repository)  
**User:** Same as installer (e.g., `denis`)  
**Group:** `docker` (required to access Docker daemon via socket)

### 4. Environment

**ENV Variables in `.env`:**
- `SYSTEM_RESET_MODE=hook` — enables webhook mode
- `SYSTEM_RESET_HOOK_URL=http://host.docker.internal:9999/restart-services` — backend target

## Installation

### Option 1: Manual (no systemd)

Run the service directly:

```bash
./scripts/restart-service-hook.py
# or through the main entry point:
./colloc_service.sh
```

Monitor logs:

```bash
tail -f logs/service.log
```

### Option 2: systemd Service (Recommended)

After running `./install.sh`, install as system service:

```bash
# Check that colloc.service was created:
cat colloc.service

# Install system-wide:
sudo cp colloc.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable colloc.service
sudo systemctl start colloc.service

# Verify:
sudo systemctl status colloc.service
journalctl -u colloc.service -f
```

Check logs via journal:

```bash
# Recent logs:
journalctl -u colloc.service -n 50

# Follow in real-time:
journalctl -u colloc.service -f

# Today's logs:
journalctl -u colloc.service --since today
```

Or local logs:

```bash
tail -f logs/service.log
```

## Testing

### 1. Service Running

Check if service is listening:

```bash
curl -s http://127.0.0.1:9999/restart-services -X POST \
  -H "Content-Type: application/json" \
  -d '{"source": "manual-test"}' | jq .
```

Expected response:

```json
{
  "status": "success",
  "message": "Services restarted successfully",
  "timestamp": 1713457821.234567
}
```

### 2. Web UI Button

Navigate to Web UI, click "Restart system" → should see success log message.

Check logs:

```bash
tail -f logs/system.log | grep "reset"
tail -f logs/restart-hook.log
```

### 3. Troubleshooting

**Service not listening:**

```bash
# Check if process is running:
ps aux | grep restart-service-hook.py

# Check port:
lsof -i :9999  # requires root

# Check logs:
tail -f logs/restart-hook.log
tail -f logs/service.log
```

**Permission denied errors:**

Ensure user is in docker group:

```bash
groups
# Output should include: docker

# If not:
sudo usermod -aG docker $USER
newgrp docker
```

**Systemd service won't start:**

```bash
sudo journalctl -u colloc.service -n 50 -p err
sudo systemctl status colloc.service --full
```

## Lifecycle

### Start

**Manual:**
```bash
./colloc_service.sh &
# or
python3 scripts/restart-service-hook.py &
```

**Systemd:**
```bash
sudo systemctl start colloc.service
```

### Stop

**Manual (if running in foreground):**
```bash
# Press Ctrl+C
# or
kill <PID>
```

**Systemd:**
```bash
sudo systemctl stop colloc.service
```

### Reload (restart)

**Manual:**
```bash
pkill -f restart-service-hook
sleep 1
./colloc_service.sh &
```

**Systemd:**
```bash
sudo systemctl restart colloc.service
```

### Check Status

**Systemd:**
```bash
sudo systemctl status colloc.service
```

**Manual:**
```bash
ps aux | grep colloc_service
ps aux | grep restart-service-hook
```

## Logs

### Locations

- `logs/service.log` — main service logs (local file)
- `logs/restart-hook.log` — hook listener logs (local file)
- systemd journal — if running as systemd service

### View

```bash
# Local logs:
tail -f logs/service.log
tail -f logs/restart-hook.log

# Systemd journal:
journalctl -u colloc.service -f
journalctl -u colloc.service --since "5 minutes ago"
```

## Requirements

### Host

- Python 3.7+
- Docker socket access (`/var/run/docker.sock`)
- User must be in `docker` group
- Port 9999 available (localhost only)

### Network (from container)

- `host.docker.internal:9999` must be reachable
- Docker's built-in name resolution handles this automatically

## Future Extensions

The `colloc_service.sh` design allows adding more host-side services:

1. Model cache cleanup daemon
2. Log rotation service
3. Metrics exporter (Prometheus)
4. Health check aggregator
5. Auto-update checker

Each service would be started as a background process with proper signal handling.

## Compatibility

- **Linux:** Full support
- **macOS (Docker Desktop):** Supported (host.docker.internal works)
- **Windows (WSL2):** Supported (host.docker.internal works)
- **Rootless Docker:** Supported (no special privileges needed)
